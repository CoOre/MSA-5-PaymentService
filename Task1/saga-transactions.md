# Task1 - Реестр транзакций Saga-оркестрации

Все транзакции процесса обработки платежа OrchestrPay с типом и, где применимо,
компенсирующей транзакцией.

## Типы транзакций

- **Компенсируемая** - до pivot-точки, её эффект можно откатить компенсацией.
- **Необратимая (pivot)** - точка невозврата, после успеха откат невозможен.
- **Повторяемая** - идемпотентна, безопасно повторяется до успеха (retry).

## Реестр транзакций

| Транзакция | Описание | Тип | Компенсация |
| --- | --- | --- | --- |
| `CREATE_PAYMENT` | Создание записи о платеже и старт экземпляра Saga (состояние - в PostgreSQL). | компенсируемая | `CANCEL_PAYMENT` |
| `DEBIT_FUNDS` | Списание/резервирование суммы заказа со счёта клиента. | компенсируемая | `REFUND_FUNDS` |
| `FRAUD_CHECK` | Антифрод-проверка (внутренние правила + AML/GeoRisk) с таймаутами, retry и кэшем в Redis. Возвращает `ALLOW`/`DENY`/`MANUAL`. | повторяемая | - (read-only, нет эффекта) |
| `AWAIT_MANUAL_REVIEW` | Ожидание решения оператора (до 20 мин); по cut-off-time - разрешено по умолчанию. | повторяемая | - (только ожидание) |
| `TRANSFER_TO_COUNTERPARTY` | Перевод средств контрагенту. **Pivot**: деньги ушли внешнему получателю, откат в рамках Saga невозможен. | необратимая (pivot) | - (пост-фактум возврат - отдельная Saga) |
| `COMPLETE_PAYMENT` | Финализация платежа в статус `COMPLETED`. | повторяемая | - |
| `NOTIFY_CLIENT` | Уведомление клиента о статусе (успех/отказ/возврат). | повторяемая | - |
| `NOTIFY_SECURITY` | Уведомление службы безопасности при подозрительной операции. | повторяемая | - |

## Компенсирующие транзакции

Компенсации **повторяемы** и идемпотентны - они обязаны дойти (деньги не должны "зависнуть").

| Транзакция | Описание | Тип | Компенсирует |
| --- | --- | --- | --- |
| `REFUND_FUNDS` | Автоматический возврат списанных средств клиенту (при `DENY`, сбое до pivot или пост-фактум фроде), без участия поддержки. | повторяемая | `DEBIT_FUNDS` |
| `CANCEL_PAYMENT` | Откат статуса в `CANCELLED`/`FAILED`, освобождение резерва, закрытие Saga. | повторяемая | `CREATE_PAYMENT` |

## Ключевые моменты

- **Порядок компенсаций** обратный: сначала `REFUND_FUNDS`, затем `CANCEL_PAYMENT`.
- **Pivot** - `TRANSFER_TO_COUNTERPARTY`; все компенсируемые шаги идут до неё, поэтому
  при отказе антифрода возможен мгновенный возврат.
- **cut-off-time** - если проверки не ответили в срок, платёж проводится по умолчанию
  (цена отклонения выше цены проведения).
- **Пост-фактум фрод** после pivot обрабатывается отдельной Saga возврата, а не
  компенсацией текущей.
